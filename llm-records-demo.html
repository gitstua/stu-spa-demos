Tiny LLM Record Processor

Paste one record per line using date|group|body|recipient. A small in-browser model loads on demand, classifies each record, estimates confidence, and suggests whether the recipient should be rerouted.

Valid recipients: stu, bob, sally.

Input Records

Each non-empty line must contain 4 fields: YYYY-MM-DD|group|body|recipient.

    How this page works: The model is intentionally small and local: keyword weights are loaded into memory, then each record body is tokenized and scored across intent classes (task, billing, scheduling, incident, social). A second lightweight scorer suggests the best recipient (stu, bob, sally) based on body/group keywords. All page state (input text + filter) is persisted in the URL hash so this demo is shareable with no localStorage, cookies, or backend.
